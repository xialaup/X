初始化本地仓库
‌创建本地仓库‌：在项目目录执行 git init 初始化仓库。 ‌


‌配置全局用户名和邮箱‌：执行 git config --global user.name "你的用户名" 和 git config --global user.email "你的邮箱地址"（可选）。 ‌


添加远程仓库地址
‌创建远程仓库‌：在云端平台（如 GitHub、 Gitee 、 GitCode 等）新建仓库。 ‌

‌关联本地与远程仓库‌：执行 git remote add origin 远程仓库地址（如 git remote add origin https://gitee.com/your_username/your_project）。 ‌

git remote add origin https://cnb.cool/junchao.tang/TV.git

推送代码到远程仓库
‌提交本地代码‌：使用 git add . 添加文件到暂存区，git commit -m '描述信息' 完成本地提交。 ‌

git branch -m main master 这条命令会将当前分支（这里是main）重命名为master

‌推送到远程仓库‌：执行 git push -u origin 本地分支名（如 git push -u origin main）。若出现 [rejected] 错误，可尝试添加 --allow-unrelated-histories 参数（如 git push -u origin master --allow-unrelated-histories）。 ‌


常见问题处理
‌非快速转发错误‌：若推送失败提示 non-fast-forward，需先执行 git pull --allow-unrelated-histories origin 本地分支名 合并代码后再重试。 ‌


‌权限问题‌：检查是否已登录正确的远程仓库地址，确认是否为可读写权限。 ‌

配置 SSH 公钥（可选）
若使用 SSH 方式推送，需在云端平台生成 SSH 公钥并配置到本地 \~/.ssh/config 或通过 git config --global url."ssh://<仓库地址>" 设置。 ‌

后期更新重复以下步骤：

git add .

git commit -m 'Enjoy享受'

git push -u origin main或master
